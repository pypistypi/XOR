# XOR File Modifier

Программа на C++/Qt под Windows. Берёт файлы из указанной папки по маске и для каждого создаёт новый файл, где каждый байт проксорен с 8-байтным ключом. Ключ повторяется по кругу до конца файла. Если прогнать получившийся файл ещё раз с тем же ключом — вернётся исходный файл, потому что XOR обратим.

Главная сложность тут — обрабатывать большие файлы (до 10 ГБ) и не вешать при этом интерфейс. Поэтому работа с файлами вынесена в отдельный поток, файл читается кусками по 1 МБ, и между кусками можно поставить паузу или остановиться.

## Сборка и запуск

Проще всего открыть `XorFileModifier/CMakeLists.txt` в Qt Creator, выбрать комплект с Qt 6 и MinGW и нажать Run.

Из консоли тоже собирается:

```
cd XorFileModifier
cmake -S . -B build -G Ninja -DCMAKE_PREFIX_PATH="C:/Qt/6.11.1/mingw_64"
cmake --build build
```

Собирал и проверял на Windows 11, Qt 6.11.1, MinGW (g++ 13.1), CMake 3.30.

## Как пользоваться

После запуска заполняем поля на форме:

- Маска — какие файлы брать, например `*.txt`. Можно несколько через `;` или `,`.
- Удалять входные файлы — удалять ли исходники после обработки.
- Папка поиска — откуда берём файлы (рядом кнопка «Обзор»).
- Папка сохранения — куда складываем результат.
- Перезапись / Счётчик — что делать, если файл с таким именем уже есть. Счётчик добавит к имени (1), (2) и так далее.
- Разовый / По таймеру — пройти папку один раз или периодически её опрашивать. Для таймера становится активным поле периода в секундах.
- Ключ — 8 байт в hex, ровно 16 символов, например `1234567890ABCDEF`.

Дальше жмём Старт. Внизу прогрессбар и строка статуса. Кнопки Пауза/Продолжить и Стоп работают в любой момент.

Чтобы проверить, что всё считается правильно: прогнать файлы из одной папки во вторую, а потом результат с тем же ключом прогнать в третью папку. Файлы в третьей папке должны совпасть с исходными.
